## Dev Notes

### Architecture Reference

This story implements the folder structure defined in:
- **Primary Source:** `docs/architecture/source-tree.md` (complete monorepo structure)
- **Supporting:** `docs/architecture.md` Section 11 (Unified Project Structure)

### Key Structural Principles

**Monorepo Benefits:**
- Atomic commits across services
- Simplified dependency management
- Easier refactoring across boundaries
- Single CI/CD pipeline

**Clean Architecture (Backend Services):**
```
[ServiceName]/
├── [ServiceName].API/           # Presentation layer (Controllers, DTOs, Middleware)
├── [ServiceName].Core/          # Domain/Business logic (Entities, Interfaces, Services)
└── [ServiceName].Infrastructure/ # Infrastructure (Data access, External services, Messaging)
```

**Dependency Direction:**
- API layer → Core layer
- Infrastructure layer → Core layer
- Core layer → No dependencies (domain-centric)

### .gitignore Configuration

Must include rules for:
- .NET: bin/, obj/, *.user, *.suo, .vs/
- Node.js: node_modules/, dist/, .env, .env.local
- IDE: .vscode/, .idea/, *.swp
- OS: .DS_Store, Thumbs.db
- Build artifacts: *.dll, *.exe, *.pdb (except in specific cases)

### .editorconfig Configuration

Standards from `docs/architecture/coding-standards.md`:
- **C# files (.cs, .csproj)**: 4 spaces, UTF-8
- **TypeScript/JavaScript (.ts, .tsx, .js, .jsx)**: 2 spaces, UTF-8
- **YAML/JSON (.yml, .json)**: 2 spaces, UTF-8
- **Markdown (.md)**: 2 spaces, UTF-8
- End of line: LF
- Insert final newline: true
- Trim trailing whitespace: true

### README.md Content Guidelines

Each README.md should contain:
1. **Folder name as title**
2. **Purpose**: Brief 1-2 sentence description
3. **Contents**: List of subfolders/key files (if applicable)
4. **References**: Links to relevant architecture docs
